# CS2 自拍杆 DLL 使用说明

## 路径

- DLL 工程：`E:\codex项目\codex-自拍杆\selfiestick\native_dll\selfiestick_hlae.sln`
- 构建脚本：`E:\codex项目\codex-自拍杆\selfiestick\build_dll.bat`
- 运行 DLL：`E:\codex项目\codex-自拍杆\selfiestick\bin\selfiestick_hlae.dll`
- 热键提示：`E:\codex项目\codex-自拍杆\selfiestick\selfiestick_bind.cfg`

## 运行入口

运行期只有一条路径：

1. 开发期编译 DLL
   ```bat
   E:\codex项目\codex-自拍杆\selfiestick\build_dll.bat
   ```
2. 在 HLAE 控制台加载 DLL
   ```cfg
   mirv_loadlibrary "E:\codex项目\codex-自拍杆\selfiestick\bin\selfiestick_hlae.dll"
   ```
3. 看到下面日志后，说明 DLL 已经进入工作状态
   ```text
   [selfiestick] ready - press Ins for panel, F8 toggle, F9 lock, F10 follow
   [selfiestick] SetUpView callback active
   ```

## 当前真实功能

- 运行期不注册自定义控制台命令。
- 运行期不使用 `mirv_script`。
- 运行期不启动外部 WPF、Named Pipe、浏览器 UI 或 bat。
- 唯一 UI 是 DLL 弹出的独立控制窗口，可拖动、可缩放。
- 面板开关键默认是 `Ins`，但现在可以在窗口里修改，并会写入 `selfiestick.ini` 保留到下次加载。
- 枪械绑定视角在开火时会对枪口抖动做稳定处理，减少镜头跟着枪身一起抖动。
- 当观战目标使用 `AWP` 开镜时，自拍杆视角保持原有构图和 FOV，不跟随开镜缩放。

## 当前支持范围

当前枪管自拍杆只对下面 4 把武器做了实装验证：

- `AK-47`
- `M4A4`
- `M4A1-S`
- `AWP`

如果当前观战目标的主动武器不在这 4 把里，DLL 不会覆写镜头，状态区会显示：

```text
weapon attachment unsupported
```

## 相机模型

当前相机不再使用旧的：

- `weapon_hand_r + 世界坐标 X / Y / Z`

当前相机固定改为：

- 枪管锚点：主动武器实体的 `muzzle_flash`
- 后视目标：`upperBodyTarget = lerp(renderEyeOrigin, weapon_hand_r, 0.30)`
- 前视方向：沿枪管前向
- 相机位置：`枪管锚点 + 步枪基础构图 + 用户 R / B / U trim`
- 开火稳定：对枪口局部位移做平滑稳定，减轻连发或单发时的瞬时抖动
- `AWP` 开镜：自拍杆开启时不接管开镜缩放，保持未开镜时的视野值

其中：

- `R` = Right，向枪右侧微调
- `B` = Back，向枪后方微调
- `U` = Up，向枪上方微调

## 正确操作手册

### 1. 先进入正确场景

要让自拍杆真正生效，必须同时满足：

- DLL 已经通过 `mirv_loadlibrary` 注入到当前 `CS2` 进程
- 当前处于观战链路
- 当前存在有效观战目标
- 当前目标的主动武器属于 `AK-47 / M4A4 / M4A1-S / AWP`
- 主动武器存在 `muzzle_flash`
- 目标玩家存在 `weapon_hand_r`

只要任意一条不满足，DLL 就不会覆写镜头，也不会走兜底路径。

### 2. 呼出面板

- 在 `CS2` 窗口获得焦点时按当前面板按键（默认是 `Ins`）
- 第一次按下：显示面板
- 第二次按下：隐藏面板

### 3. 启用自拍杆

有两种方式：

- 面板显示时，点击 `SELFIESTICK ON / OFF`
- 面板隐藏时，按 `F8`

注意：

- `F8 / F9 / F10` 只在面板隐藏时生效
- 面板显示时，键鼠输入由面板接管

### 4. 调整构图

面板里的三轴已经不是世界坐标：

- `R`：枪右侧 trim
- `B`：枪后方 trim
- `U`：枪上方 trim

默认值现在是：

```text
R=0 / B=0 / U=0
```

因为步枪基础构图已经内置，输入框只负责微调。

### 5. 切换朝向

- `左侧自拍`：从枪械左侧自拍角色
- `右侧自拍`：从枪械右侧自拍角色
- `向前看`：沿枪口方向看向前方

切换朝向只改变看向语义，不改变：

- 跟随 / 锁定逻辑
- 枪管锚点
- 用户 trim

### 6. 切换目标模式

- `FOLLOW`：跟随当前观战目标
- `LOCK CURRENT`：锁定当前观战目标
- `CLEAR`：清除锁定并回到跟随

面板隐藏时热键仍然可用：

- `F9`：锁定当前目标
- `F10`：回到跟随

## 状态区怎么读

状态区显示的是当前运行事实：

- `Enabled`
- `Look`
- `Target mode`
- `Locked handle`
- `Current target`
- `Barrel attach`
- `Trim R/B/U`
- `Last override`
- `Reason`

重点看这三项：

- `Barrel attach: muzzle_flash / missing`
  - 说明主动武器枪管 attachment 当前不可用
- `Last override: failed`
  - 说明本帧没有成功覆写镜头
- `Reason: ...`
  - 这里就是唯一真实失败原因

## 最小排错顺序

如果面板能呼出，但自拍杆没有效果，按这个顺序看：

1. 重新加载 DLL 后确认控制台出现：
   - `[selfiestick] ready - press Ins for panel, F8 toggle, F9 lock, F10 follow`
2. 进入观战画面后确认出现：
   - `[selfiestick] SetUpView callback active`
3. 按 `Ins` 打开面板
4. 先看状态区里的 `Reason`
5. 再按 `F8` 或点启用按钮，把 `Enabled` 切到 `on`
6. 如果仍然没有效果，优先排查这些真实失败原因：
   - `no observer target`
   - `observer target unresolved`
   - `active weapon missing`
   - `weapon attachment unsupported`
   - `attachment missing`
   - `weapon_hand_r missing`
   - `weapon forward degenerate`
   - `weapon right axis degenerate`
   - `weapon up axis degenerate`
   - `camera direction degenerate`

## 当前固定行为

- 主动武器识别固定靠 RTTI 类名
- 当前支持武器固定为 `AK-47 / M4A4 / M4A1-S / AWP`
- 枪管 attachment 固定为 `muzzle_flash`
- 上半身参考 attachment 固定为 `weapon_hand_r`
- 没有 attachment fallback
- 没有第二套相机模型
- 没有运行期控制台命令
- 没有外部 UI 进程

## 默认热键

```cfg
Ins -> 默认显示 / 隐藏控制窗口（可在面板内修改）
F8  -> 启用 / 关闭自拍杆
F9  -> 锁定当前观战目标
F10 -> 回到跟随模式
```
