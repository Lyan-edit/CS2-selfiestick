# CS2 自拍杆 DLL 操作手册

## 发布目录内容

中文发布目录包含：

- `Lyan_CS2自拍杆.dll`
- `操作手册.md`
- `selfiestick_bind.cfg`

实际运行只需要 DLL。手册和 cfg 是说明文件。

DLL 已静态链接 MSVC 运行库，用户不需要额外安装 Visual C++ Redistributable 就可以通过 HLAE 加载。

## 支持版本

当前发布包标记支持：

- HLAE：`2.190.2`
- Counter-Strike 2：Steam app `730`，buildid `23669931`

CS2 和 HLAE 经常更新。如果后续游戏 schema、实体布局或 `SetUpView` 变化，DLL 可能会主动阻塞，并在面板里显示原因。

## 加载 DLL

1. 把整个 `zh-CN` 文件夹复制到稳定位置。
2. 启动 HLAE `2.190.2`，进入 CS2，并打开 demo 观战。
3. 在 HLAE 控制台执行：

```cfg
mirv_loadlibrary "<Lyan_CS2自拍杆.dll 的绝对路径>"
```

示例：

```cfg
mirv_loadlibrary "D:\tools\selfiestick\zh-CN\Lyan_CS2自拍杆.dll"
```

加载成功后，按 `Ins` 打开 ImGui 面板。

如果 HLAE 提示 `mirv_loadlibrary failed`，请先完整重启 HLAE/CS2，并确认绝对路径正确。当前 release DLL 不再依赖 `MSVCP140.dll`、`VCRUNTIME140.dll`、UCRT `api-ms-win-crt-*` DLL，也不再把 `D3DCOMPILER_47.dll` 作为加载时依赖。

当前 release DLL 在 SetUpView hook 不需要 schema offsets 时，也不会在启动阶段主动执行重 schema 解析。发布包里的 `selfiestick.ini` 默认关闭调试日志；只有需要发诊断日志时，才建议手动设置 `[Debug] TraceEnabled=1`。

如果 DLL 同目录已经有旧版生成的 `selfiestick.ini`，测试新版前请删除它，或确认 `[Debug] TraceEnabled=0`。

推荐设置速度：道具自拍调机位时，建议先通过 CS2 demo UI / HLAE 播放控制把 demo 降到 `0.1x` 倍速；如果当前游戏控制台可用，也可以执行：

```cfg
demo_timescale 0.1
```

## 热键

- `Ins`：默认显示 / 隐藏面板
- `F8`：启用 / 关闭自拍杆
- `F9`：锁定当前观战目标
- `F10`：目标模式回到跟随

面板开关键可以在面板里修改，并保存到 DLL 同目录的 `selfiestick.ini`。

## 人物自拍模式

在 `相机模式` 里选择 `人物自拍`，使用原来的人物自拍杆。

主要控制：

- `左侧自拍`：从目标左侧构图自拍
- `右侧自拍`：从目标右侧构图自拍
- `向前看`：前视模式
- `跟随`：每帧使用当前观战目标
- `锁定当前`：锁定当前目标探员
- `清除锁定`：清除目标锁定
- `人物 右 / 后 / 上`：人物相机位置微调，可以输入数字，也可以使用标注为 `右`、`后`、`上` 的滑轨
- `人物 俯仰 / 偏航 / 翻滚`：人物相机角度微调，可以输入数字，也可以使用标注为 `俯仰`、`偏航`、`翻滚` 的滑轨

内置构图会先应用，人物 `右 / 后 / 上` 和 `俯仰 / 偏航 / 翻滚` 都是在原本左侧自拍、右侧自拍、向前看视角上的额外微调。

## 道具自拍模式

在 `相机模式` 里选择 `道具自拍`，相机会跟随当前目标探员投出的飞行道具。

推荐工作流：先把 demo 播放速度调到 `0.1x`，再跟随或锁定目标探员；等道具候选出现后，单个道具会自动锁定，多个道具则在 `最近道具` 中手动点 `锁定`；之后调 `X / Y / Z` 和 `俯仰 / 偏航 / 翻滚`。构图稳定后，再按需要提高 demo 速度。

支持道具：

- 烟
- 火，包含 molotov / incendiary
- 雷
- 闪
- 诱饵

行为规则：

- 目标探员仍然由 `跟随 / 锁定当前` 决定。
- 只接受当前目标探员 pawn 或 controller 拥有 / 投出的飞行道具。
- 如果当前只有 1 个有效飞行道具，并且没有锁定道具，会自动锁定。
- 如果同时有多个有效飞行道具，不会自动抢锁，需要在 `最近道具` 中手动点 `锁定`。
- `清除道具` 会清除当前道具锁定。如果下一帧仍然只有 1 个有效道具，可以再次自动锁定。
- 道具相机回看方向优先使用实际投掷者，所以切换观战目标或 T/CT 方视角不一致时，朝向仍按投掷者方向。
- 烟雾弹飞行实体消失后会短暂保留最后镜头；其他道具飞行实体结束后正常结束。

道具控制：

- `道具 X / Y / Z`：道具相机局部偏移
- `X / Y / Z` 滑轨：范围 `-64` 到 `64`，用于快速调整道具相机位置
- `俯仰 / 偏航 / 翻滚`：角度微调，可以输入数字，也可以使用滑轨
- `俯仰 / 偏航 / 翻滚` 滑轨：范围 `-45` 到 `45` 度，灵敏度适中，适合快速避开穿模
- `截图风格`：一键写入统一截图参数：`X=-8`、`Y=-22`、`Z=4`、`俯仰=1`、`偏航=0`、`翻滚=0`
- `最近道具`：显示候选道具、是否飞行中、句柄、投掷者句柄、帧龄、类型和锁定按钮

道具相机使用平滑预测锚点，减少慢放和正常速度下的一跳一跳、上下抽搐和卡顿感。

## 状态与排错

运行状态区会显示：

- 启用状态
- 相机模式
- 朝向
- 面板按键
- 目标模式
- 锁定目标 / 当前目标
- 道具锁定与候选
- 最近覆写结果
- 失败原因

调试日志默认关闭。需要排查问题时，可以在 DLL 同目录的 `selfiestick.ini` 里设置：

```ini
[Debug]
TraceEnabled=1
```

完整重启 HLAE/CS2 后，日志会写在加载 DLL 的同目录：

```text
selfiestick_trace.log
```

如果替换 DLL 后仍加载旧版本，请完整重启 HLAE 和 CS2。Windows 会把已加载 DLL 保留在进程里，单纯覆盖文件不一定会立即生效。

## 安全说明

本 DLL 只用于 HLAE demo / 影视工作流，不用于匹配、实战或任何反作弊保护服务器。使用 HLAE 或注入工具进入 VAC 服务器存在风险。
